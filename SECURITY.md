# Security Policy

## Supported Versions

The `master` branch and latest tagged release are actively supported.
Older tags may receive best-effort fixes only.

## Reporting a Vulnerability

Do not open public issues for suspected vulnerabilities.

Report privately by emailing: `saagar210@gmail.com`

Include:

- affected commit/tag
- impact summary
- reproduction steps
- proof-of-concept or logs (redacted)

## Response Targets

- Initial acknowledgment: within 2 business days
- Triage decision: within 5 business days
- Critical/High remediation target: within 14 calendar days
- Medium remediation target: next planned hardening cycle

## Disclosure Process

1. Confirm and classify severity.
2. Develop and test fix on a private branch.
3. Release patched version.
4. Publish coordinated advisory summary after patch availability.

## Security Ownership

- Primary owner: `@saagar210`
- Backup owner: repository admin
- Escalation fallback: repository admin
